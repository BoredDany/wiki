Requirements

Funcional: El sistema debe permitir al arrendatario crear, editar, eliminar y actualizar propiedades para arrendar. El sistema debe permitir a los arrendatarios solicitar el arrendamiento de una propiedad. El sistema debe permitir al arrendador revisar y responder (aceptar o cancelar) solicitudes de arrendamiento de propiedades. El sistema debe permitir al arrendador verificar los pagos de los arrendatarios. El sistema debe permitir al arrendador calificar y comentar la experiencia de un arrendamiento. El sistema debe permitir al arrendatario buscar propiedades disponibles filtrándolas por ubicación geográfica. El sistema debe permitir al arrendatario pagar un arrendamiento una vez sea aceptado por el arrendador. El sistema debe permitir al arrendatario calificar propiedades y experiencias de arrendamiento. No Funcional: El sistema debe guardar los datos en una base de datos de MySQL. El sistema debe utilizar el framework de SpringBoot y thymeleaf. El sistema debe utilizar la API de JPA para las operaciones con la base de datos. El sistema debe contar con interfaces de usuario intuitivas y fáciles de usar. El sistema debe integrar un sistema de autenticación y autorización para gestionar los permisos de los usuarios.
